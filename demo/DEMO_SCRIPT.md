# 60-Second Demo Video Script

**Industrial Audio RAG System - Demo Script**

## 🎬 Video Structure (60 seconds)

### Opening (0-10 seconds)
**[Screen: Clean browser with HuggingFace Space loaded]**

**Voiceover:** 
> "Meet Industrial Audio RAG - an AI system that answers questions about industrial audio data using state-of-the-art retrieval-augmented generation."

**Actions:**
- Show the main interface with title and description
- Highlight the clean, professional UI

### Problem Statement (10-15 seconds)
**[Screen: Show sample questions dropdown]**

**Voiceover:**
> "Instead of manually analyzing thousands of audio clips, engineers can now ask natural language questions."

**Actions:**
- Click through sample questions quickly
- Show the variety of queries possible

### Demo Query 1 - Simple (15-30 seconds)
**[Screen: Type and submit first query]**

**Query:** "Which bearing clips show dominant frequency above 900 Hz?"

**Voiceover:**
> "Let's ask about frequency analysis. I'll type: 'Which bearing clips show dominant frequency above 900 Hz?'"

**Actions:**
- Type query smoothly
- Click "Analyze" button
- Show progress indicator
- Highlight the structured answer when it appears
- Point out performance metrics (response time, sources found)

### Demo Query 2 - Complex (30-45 seconds)
**[Screen: Type and submit second query]**

**Query:** "What are the differences between normal and abnormal bearing signatures?"

**Voiceover:**
> "Now a more complex question about bearing condition analysis."

**Actions:**
- Clear previous query
- Type new query
- Submit and wait for response
- Show the detailed answer with technical insights
- Scroll to sources section to show attribution

### Technical Highlights (45-55 seconds)
**[Screen: Show system information and metrics]**

**Voiceover:**
> "The system uses vector embeddings to find relevant audio clips, then generates answers using a language model. All with full source attribution and real-time performance metrics."

**Actions:**
- Show system information panel
- Highlight performance metrics
- Scroll through sources section
- Show the GitHub link

### Closing (55-60 seconds)
**[Screen: Show GitHub repository link]**

**Voiceover:**
> "Ready for production deployment with Docker, Kubernetes, and comprehensive monitoring. Full source code available on GitHub."

**Actions:**
- Click GitHub link (prepare to cut before it loads)
- Show clean end screen with logo/title

## 📋 Pre-Recording Checklist

### Technical Setup
- [ ] Ensure demo API is running and responsive
- [ ] Test all sample queries work correctly
- [ ] Clear browser cache and cookies
- [ ] Set up clean browser window (no bookmarks/extensions visible)
- [ ] Test screen recording software
- [ ] Ensure stable internet connection

### Content Preparation
- [ ] Practice typing queries smoothly
- [ ] Time each section to ensure 60-second total
- [ ] Prepare backup queries in case of API issues
- [ ] Test all UI interactions
- [ ] Verify system info shows correctly

### Recording Environment
- [ ] Quiet environment for voiceover
- [ ] Good lighting for screen recording
- [ ] Stable computer performance
- [ ] Multiple takes planned

## 🎤 Voiceover Tips

### Tone
- Professional but enthusiastic
- Clear enunciation
- Moderate pace (not rushed)
- Confident delivery

### Emphasis Points
- "Retrieval-Augmented Generation"
- "Real-time performance metrics"
- "Production ready"
- "Full source attribution"

### Timing Cues
- Sync voice with actions on screen
- Pause for API response times
- Match excitement to demo moments

## 🎬 Video Production Notes

### Recording Settings
- **Resolution**: 1920x1080 minimum
- **Frame Rate**: 30fps
- **Format**: MP4 for web compatibility
- **Audio**: 44.1kHz, clear recording

### Editing Points
- Cut any API delays longer than 2 seconds
- Smooth transitions between queries
- Clean up any UI glitches
- Add subtle background music (optional)

### Export Settings
- File size under 50MB for easy sharing
- High quality for professional presentation
- Multiple formats (MP4, WebM) for compatibility

## 🔄 Backup Plans

### If API is slow/down:
- Use pre-recorded responses
- Mock responses with static content
- Focus more on UI and features

### If demo breaks:
- Have backup video of working system
- Screenshots of key moments
- Static slides with key points

### If timing runs over:
- Priority: problem statement + one demo
- Cut: system information details
- Keep: GitHub link and closing

This script ensures a professional, engaging demo that showcases the system's capabilities effectively within the 60-second constraint.
